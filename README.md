# Nippo System

Nippo Systemは、作業中のパソコン画面（OCR）や、キーボード・マウスの入力履歴を背後で「超軽量」に記録し、それらの生データを自動で結合（意味付け）して**「AI向けの日報用データ（YAML）」を全自動で生成**する画期的な常駐型システムです。

## ✨ 主な機能とパイプライン

システムは独立したモジュールのバケツリレー形式（パイプライン）で動作します。

### 1. データの収集 (Input Monitor & Screen OCR)
各モジュールが裏側で独立して生データ（Raw Streams）を集め、TSVファイルとして書き出します。
- **超高速OCR**: 新開発のWinRT EngineやTesseract（インメモリレジデント化）を用いることで、起動時のオーバーヘッドをゼロにし、ノートPCでも快適に裏側で動作し続ける負荷最適化を実現しました。
- **入力監視**: キーボードの打鍵タイミングと、マウスのクリック座標を記録します。

### 2. コンテキスト結合・意味付け (Transformer)
システム最大のキモとなる部分です。バラバラに保存された生データを統合します。
- **マウスクリックの意味付け**: マウスの「(X,Y)座標」と、OCRが読み取った「文字の(X,Y)座標」を物理的に重ね合わせることで、**「ユーザーがどのボタンや文字をクリックしたか」**を特定し、意味を抽出します。
- **タイピングの意味付け**: 生のキー入力から、バックスペースなどの操作を加味して最終的に確定された文章を復元します。

### 3. 日報AI用YAMLへの高度圧縮 (Aggregator)
意味付けされたデータから、ユーザーの活動履歴や空白時間（アイドル時間）、タイピング密度を計算し、AI（LLM）が最も読み込みやすい極限まで圧縮された1つの `YAML` ファイルを出力します。

### 4. モダンなGUIダッシュボード (`gui_app.py`)
これらすべての裏方作業を、洗練された**1つのGUI画面**で管理します。
- システムトレイ（タスクバー右下）に常駐し、普段の作業の邪魔になりません。
- PC立ち上げ時の「スタートアップ全自動起動」に標準対応。
- **「📝 作業終了（YAML作成）」ボタン** のワンクリックで、TransformerとAggregatorを全自動で回し、日報用データの書き出し〜確認までを一瞬で完了させます。

---

## 🚀 使い方

1. `python nippo_system/gui_app.py` でダッシュボードを起動します。
2. （必要であれば）画面上のスイッチで「スタートアップ自動起動」をONにします。
3. デスクトップに表示されている画面の「最小化して常駐 (トレイへ)」を押し、普段通りにPC作業を行います。（OCRや入力監視が背後で超軽量に稼働し続けます）
4. 一日の終わりにトレイアイコンから画面を呼び出し、**「📝 作業終了（YAML作成）」** ボタンを押します。
5. `reports/` フォルダ内に生成された `nsl_story_YYYY-MM-DD.yaml` が自動で開かれます。これをAIに渡すことで、高度な日報が仕上がります！
