# 🏗️ nippo_system フォルダ構成と設計方針

## 1. ディレクトリ構成
開発環境 (`c:\Users\y86as\Nippo\nippo_system`) の主要な構成です。

```text
nippo_system/
├── main.py            # [Manager] 全プロセスを個別に起動・管理
├── core/              # [Core] 共通設定と共有データ
├── ocr/               # [Feature] 画面解析モジュール (Stage 1)
├── input_monitor/     # [Feature] 入力活動監視モジュール (Stage 1)
├── transformer/       # [Stage 2] 生データを意味のある情報に変換 (NSL生成)
├── reporter/          # [Stage 3] セマンティックデータを集計・物語化 (YAML生成)
├── ai_prompts/        # [Stage 4] LLM用プロンプトテンプレートの管理
├── raw_streams/       # [Data] 収集された生データ (TSV)
├── reports/           # [Output] 生成された日報・YAMLの出力先
└── data/              # [Internal] SQLite DBやモデルファイル
```

## 2. 疎結合 (Decoupling) と パイプライン
本システムは、各機能が独立して動作し、ファイル（Data Streams）を介してバトンを繋ぐデザインを採用しています。

- **Stage 1: Collection**: `ocr`, `input_monitor` が非同期に生データを収集。
- **Stage 2: Semantic Transformation**: `transformer` が生データに意味（何をクリックしたか等）を付加。
- **Stage 3: Narrative Aggregation**: `reporter` が時系列の「仕事の物語」として集計。
- **Stage 4: AI Reporting**: `ai_prompts` を活用し、LLMが高密度な日報を生成（現在は手動投稿フロー）。
