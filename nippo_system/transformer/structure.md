# Transformer モジュールの構造

このモジュールは、記録された生の活動ストリームに対して **意味付け（Semantic Annotation）** を行う役割を担います。

```text
transformer/
├── main_transformer.py  # 中央司令塔（工場ライン）
├── annotators/          # 意味付けの専門家（モジュール）
│   ├── click_map.py     # クリック座標 ∩ OCR = 何をクリックしたかの特定
│   ├── key_logic.py     # 生キーログ ➔ 入力の意図（タイピング内容と推敲の統計）
│   └── context.py       # ウィンドウタイトル ➔ 作業カテゴリの判定
└── semantic_data/       # 意味付けが完了した成果物（JSON）の保管場所
    ├── semantic_clicks_YYYY-MM-DD.json
    ├── semantic_typing_YYYY-MM-DD.json
    └── semantic_context_YYYY-MM-DD.json
```

## 仕組み
1. **Raw Streams**（生データ）を `raw_streams/` ディレクトリから読み込みます。
2. **Main Transformer** が各専門家（annotators）を順に呼び出し、単なる座標を「ボタン名」に、キーの羅列を「文章」に、ウィンドウ名を「業務コンテクスト」に変換（意味付け）します。
3. **Semantic Logs**（意味付きログ）を `semantic_data/` フォルダに出力します。これらは Stage 3（AIレポート生成）が「今日何をしたか」を理解するための直接のインプットになります。

## 関連コンポーネント
- `reporter/aggregator.py`: これらのセマンティックJSONを統合し、時系列の「物語」に編み上げます。
- `ai_prompts/`: 集計されたデータを元に AI が最高の日報を書くためのプロンプトを管理します。
